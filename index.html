Core Fundamentals

  1. Explain the CSS Box Model.
    The CSS box model consists of content, padding, border, and margin. content-box: width does NOT include padding and border. border-box: width includes padding and border (recommended). 
    div {
  width: 200px;
  padding: 20px;
  border: 10px solid;
}
    content-box total width = 260px border-box total width = 200px
  2. display:none vs visibility:hidden vs opacity:0
    display:none removes the element completely from layout. visibility:hidden hides it but keeps space. opacity:0 hides visually but element is still clickable. Example: 
    div { opacity: 0; }
    User can still interact with it.
  3. CSS specificity
    Specificity decides which CSS rule wins. Priority: Inline > ID > Class > Element Example: 
    #nav .item:hover
    Specificity = 1 ID + 2 class = (1,2,0)
  4. Units (em, rem, vw, vh, px)
    px → fixed size em → relative to parent rem → relative to root vw/vh → relative to viewport Example: 
    font-size: 2rem;
  5. Stacking context
    Controls layering of elements. Created by: - position + z-index - transform - opacity < 1 Example: 
    div {
  position: relative;
  z-index: 10;
}
  6. inline vs block vs inline-block
    inline → no width/height block → full width inline-block → supports width/height but stays inline
  7. CSS Cascade
    Order: 1. !important 2. Specificity 3. Source order Example: 
    p { color: red; }
p { color: blue; }
    Blue wins (last rule).

Flexbox & Grid

  1. justify-content vs align-items
    justify-content aligns items horizontally (main axis). align-items aligns vertically (cross axis).
  2. flex: 1 1 auto
    flex-grow:1 → expand flex-shrink:1 → shrink flex-basis:auto → default size
  3. flex properties
    grow expands, shrink reduces, basis defines initial size.
  4. Responsive grid 
    grid-template-columns: repeat(auto-fit, minmax(250px,1fr));
    Auto adjusts columns without media query.
  5. auto-fill vs auto-fit
    auto-fill keeps empty space auto-fit collapses empty tracks
  6. Grid vs Flexbox
    Grid = 2D layouts Flexbox = 1D layouts
  7. grid-template-areas
    Defines layout using names. 
    grid-template-areas:
"header header"
"sidebar main";
  8. align-content vs align-items
    align-items → items align-content → container space

5+ Years Experience (Advanced / Real-World / Scenario Based)

  1. How do you debug a complex CSS issue in production where styles are breaking randomly?
    Step-by-step approach:
    • Use browser DevTools → inspect computed styles
    • Check specificity conflicts
    • Look for overwritten styles (strikethrough rules)
    • Check loaded CSS order
    • Look for dynamic class changes (JS)
    Pro Tip: Use "Computed" tab to trace final applied styles.
  2. Explain how browser rendering works (critical rendering path).
    Steps: 1. HTML → DOM 2. CSS → CSSOM 3. Combine → Render Tree 4. Layout → position calculation 5. Paint → pixels drawn Key insight: CSS blocking delays rendering.
  3. What causes layout thrashing and how to prevent it?
    Repeated DOM read/write causes reflow. Example problem: 
    element.style.width = "100px";
element.offsetHeight;
    Fix: - Batch DOM updates - Use requestAnimationFrame
  4. Difference between reflow and repaint?
    Reflow → layout recalculation (expensive) Repaint → visual update only Example: - width change → reflow - color change → repaint
  5. How do you optimize CSS for large applications?
    - Avoid deep selectors - Use BEM or modular CSS - Minimize unused CSS - Use critical CSS Tools: DevTools coverage tab
  6. Explain critical CSS and how you implement it.
    Critical CSS = above-the-fold styles loaded first. Implementation: - Inline critical CSS - Load rest asynchronously Improves page load performance.
  7. How does contain property improve performance?
    Limits scope of layout/paint. Example: 
    .container {
  contain: layout paint;
}
    Prevents unnecessary reflows.
  8. What are compositing layers and why are they important?
    GPU layers improve performance. Triggered by: - transform - opacity Example: 
    transform: translateZ(0);
  9. Explain how you would build a scalable design system.
    - Use tokens (colors, spacing) - Component-based CSS - Naming conventions (BEM) - Documentation Example: 
    --primary-color: #4a6cf7;
  10. How would you handle theming (multiple themes)?
    Use CSS variables: 
    :root {
  --bg: white;
}
.dark {
  --bg: black;
}
  11. Why does position: fixed behave differently inside transform?
    Transform creates new containing block. So fixed becomes relative to that parent instead of viewport.
  12. How do you prevent FOUC (Flash of Unstyled Content)?
    - Load CSS early - Inline critical CSS - Avoid blocking scripts
  13. What is CSS Houdini (conceptually)?
    Allows developers to extend CSS with custom logic. Example: Custom paint API for drawing backgrounds.
  14. How do you manage CSS in large React/Next.js apps?
    Options: - CSS Modules - Styled-components - Tailwind Goal: avoid global conflicts.
  15. Explain difference between inline styles and classes in performance.
    Inline styles: - Higher specificity - Hard to override Classes: - Reusable - Better performance for large apps
  16. How would you debug a flexbox issue quickly?
    Use DevTools: - Toggle flex overlay - Check flex properties - Inspect parent vs child Common issue: missing min-width:0
  17. Why does overflow:hidden break sticky?
    Creates new scroll container → sticky stops referencing viewport.
  18. How to reduce CSS bundle size?
    - Remove unused CSS - Use PurgeCSS - Minify files
  19. Explain layering strategy in complex UI.
    Use consistent z-index scale: 
    100 = header  
1000 = modal
    Avoid random values.
  20. How do you handle cross-browser CSS issues?
    - Use autoprefixer - Test in multiple browsers - Use feature queries Example: 
    @supports (display: grid) { }
  21. Explain difference between visibility:hidden and pointer-events:none
    visibility:hidden → hidden pointer-events:none → disables interaction only
  22. How do you ensure accessibility in CSS?
    - Good contrast - Focus styles - Avoid motion (prefers-reduced-motion)
  23. How would you animate height:auto?
    Not directly possible. Workaround: - max-height trick - JS measurement
  24. Explain CSS architecture methodologies.
    - BEM - SMACSS - ITCSS Used for maintainability.
  25. How would you debug random spacing issues in layout?
    - Check margin collapse - Inspect box model - Look for inherited styles

CSS Calculation Based Questions (Beginner → Advanced)

  1. Element width = 100px, padding = 20px, border = 10px (content-box). What is total width?
    In content-box, padding and border are added outside. Calculation: 100 + (20×2) + (10×2) = 100 + 40 + 20 = 160px
  2. Same case with box-sizing: border-box
    In border-box, width includes padding and border. Total width remains: 100px Content area shrinks internally.
  3. Width = 200px, margin = 20px. Total occupied width?
    Margin adds outside space. 200 + (20×2) = 240px
  4. Parent width = 500px, child width = 50%. What is child width?
    50% of 500 = 250px
  5. font-size = 16px, element = 2em. Final size?
    2 × 16 = 32px
  6. Root font-size = 16px, element = 2rem. Final size?
    2 × 16 = 32px (rem uses root)
  7. Parent font-size = 20px, child = 2em. What is child size?
    2 × 20 = 40px
  8. Width = calc(100% - 50px), parent = 500px. Final width?
    100% of 500 = 500 500 - 50 = 450px
  9. Width = 100vw, viewport width = 1200px
    100vw = 1200px
  10. Height = 50vh, viewport height = 800px
    50% of 800 = 400px
  11. padding: 10px 20px; What is vertical and horizontal padding?
    Top & Bottom = 10px Left & Right = 20px
  12. margin: 10px 20px 30px 40px; What is left margin?
    Order: top right bottom left Left = 40px
  13. Box width = 100px, border-box, padding = 20px. What is content width?
    100 - (20×2) = 60px
  14. Grid container width = 600px, 3 columns (1fr each). Width per column?
    600 / 3 = 200px
  15. Grid: repeat(3, 1fr) + gap: 30px. Container = 630px. Column width?
    Total gap = 30×2 = 60px Remaining = 630 - 60 = 570 Each column = 570 / 3 = 190px
  16. Flex container = 600px, 3 items flex:1. Each width?
    600 / 3 = 200px
  17. Width = clamp(200px, 50%, 400px), parent = 1000px
    50% = 500px Clamp max = 400px Final = 400px
  18. Width = clamp(200px, 30%, 400px), parent = 500px
    30% = 150px → below min So result = 200px
  19. Line-height = 1.5, font-size = 20px. Actual line height?
    1.5 × 20 = 30px
  20. Element width = 300px, transform: scale(2). Visual width?
    300 × 2 = 600px (visual only, layout unchanged)
